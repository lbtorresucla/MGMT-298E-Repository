PointStack.AI — System Diagram

6-agent pipeline with conditional clarification and quality retry loop

Agent What it does Key outputs
Input Collector Receives the rough bullet and identifies the apparent role, industry, and core accomplishment. Flags any obviously missing context (level, scope, industry). Structured summary of what the bullet is about and what context is present or absent.
Bullet Diagnostic Evaluates the bullet on four dimensions: action verb strength, quantification, specificity, and impact clarity. Rates each as weak / moderate / strong. Diagnostic scorecard with specific issues to fix before rewriting.
Impact Quantifier Identifies every opportunity to add or strengthen metrics. Labels each suggestion as must ask user or can infer from context. List of specific metric types to insert (%, $, headcount, time saved, etc.) and where they belong.
User Clarification
(conditional)		 If the bullet is vague, pauses the pipeline and asks the user up to 8 targeted questions — one per missing detail — with examples to jog memory. Covers: Quantity · Type · Timeline · Outcome · Financial Impact · Scale · Efficiency Gains · Other Impact. Skipped if bullet is already specific.
MBB Rewrite Rewrites the bullet in McKinsey / Bain / BCG style. Must open with a verb from strong_resume_verbs.json (9 categories). Incorporates all user-provided context and quantification notes. Max 25–30 words. Single polished bullet: [Strong verb] + [quantified impact] + [strategic context]. Placeholders like [X%] used where data is unknown.
Hallucination Check Compares the rewritten bullet against the original. Flags any claim, number, or fact that was not present in or directly inferable from the original. Placeholders are acceptable — not hallucinations. PASS (proceeds) or FAIL (loops back to MBB Rewrite with instructions to strip invented details).
LLM-as-a-Judge Scores the rewritten bullet 1–10 across five 2-point criteria: action verb strength, quantified impact, strategic framing, conciseness, and overall MBB polish. Pass threshold is ≥ 7. Score, per-criterion breakdown, PASS / FAIL verdict, and one actionable feedback sentence. Loops back to MBB Rewrite (up to 3 attempts) if score < 7.
Finalized Output Displays the approved bullet with its score badge (green ≥ 8, orange ≥ 6, red < 6), full score breakdown, and a one-click copy button. Final polished resume bullet ready to paste into your resume.
